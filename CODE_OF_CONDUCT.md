# Code of Conduct

OVCA Core welcomes careful, respectful collaboration. Participants must engage
in good faith, critique ideas and code rather than people, respect privacy, and
avoid harassment, threats, discrimination, or publication of private material.

Maintainers may edit, hide, or reject contributions and restrict participation
when conduct harms the project or its contributors. Reports should use GitHub
private vulnerability reporting when it is enabled for this repository. If it
is unavailable, open a minimal public issue asking maintainers to provide a
private contact channel. Do not include vulnerability details, credentials,
personal data, or private paths in the issue. After publication, maintainers
should enable private vulnerability reporting. Enforcement decisions should be
proportionate, documented privately, and protect reporters from retaliation.
